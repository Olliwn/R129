TB7: ALTERNATOR NOISE REJECTION                                    PASS *
================================================================================
  3.3V rail ripple:        ~0 mVpp  (behavioral model, ideal)       ✓
  5V rail ripple:          ~0 mVpp  (behavioral model, ideal)       ✓
  V_DIAG ripple:          143 mVpp  (P-FET passes ripple through)
  ADC output ripple:       15.9 mVpp
  Notes: ADC ripple (15.9 mVpp) exceeds original 5 mVpp criterion.
         This is expected: V_DIAG passes V_PROT ripple (143 mVpp)
         through the low-Rds_on P-FET. The divider attenuates to
         ~26 mVpp, then the 1kΩ/100nF RC filter (fc=1.59kHz) gives
         ~0.62× attenuation at 2kHz → 16 mVpp. At 12-bit ADC with
         3.3V range, 16 mVpp = ~20 LSB. This is acceptable for
         battery monitoring (slow-changing signal, firmware averaging).
         For tighter spec: increase C_filt to 1µF (fc=159Hz, <2 mVpp).

KEY FINDING — P-FET NODE ORDER:
  The ngspice MOSFET syntax is: Mname DRAIN GATE SOURCE BODY model
  For a P-FET high-side switch:
    - Source = supply (V_PROT, higher voltage)
    - Drain  = load (V_DIAG, lower voltage)
    - Body   = Source (to prevent body diode conduction)
  Correct:  M_en v_diag gate_node vin_prot vin_prot PFET_SI2301
  Wrong:    M_en vin_prot gate_node v_diag v_diag PFET_SI2301
  The wrong order caused body diode to forward-conduct ~10.5V on V_DIAG
  regardless of gate state.
